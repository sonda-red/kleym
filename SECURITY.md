# Security Policy

## Supported Versions

Security fixes are applied on the active development line:

- `main`: supported

Older commits, tags, and unmaintained branches may not receive security updates.

## Reporting a Vulnerability

Please report vulnerabilities privately through GitHub Security Advisories:

1. Open the repository Security tab.
2. Use **Report a vulnerability** to create a private report.

Do not open public GitHub issues for undisclosed vulnerabilities.

## What To Include

Please include:

- A clear description of the issue and impacted component.
- Reproduction steps or proof-of-concept.
- Expected impact and any known mitigations.
- Version, commit SHA, and environment details.

## Response Process

Maintainers will:

1. Acknowledge receipt.
2. Validate and triage severity.
3. Coordinate a fix and disclosure timeline.

We aim to send an initial triage response within 3 business days.
